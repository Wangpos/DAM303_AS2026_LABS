2. Background
2.1 From Q-Table to Policy Network
In Practical 2 you stored Q-values in a table: one row per state, one column per action. This
works for small discrete state spaces (25 cells). CartPole has a continuous 4-dimensional
state space — there are infinitely many possible states, so a table is impossible. Instead, we
use a neural network that takes the state vector as input and outputs action probabilities
directly. This is called a policy network.

2.2 The CartPole Environment
CartPole-v1 is the standard benchmark for policy gradient methods. A pole is balanced on a
cart that can move left or right. The agent must keep the pole upright for as long as possible.
Property Details
Observation space 4 continuous values: cart position, cart velocity, pole angle,
pole angular velocity
Action space 2 discrete actions: 0 = push left, 1 = push right
Reward +1 for every step the pole stays upright
Episode ends Pole angle > 12 degrees, or cart moves out of bounds, or
500 steps reached
Solved threshold Average reward >= 475 over 100 consecutive episodes
2.3 Policy Gradient — REINFORCE
REINFORCE is the simplest policy gradient algorithm. The idea: run a full episode, then look
back at every action taken and ask — given the return that followed, should this action be
made more or less likely?
The algorithm in five steps:
1. Run one full episode under the current policy, collecting (state, action, reward) at
each step

2.4 PyTorch AutoDiff in 60 Seconds
PyTorch tracks all operations on tensors that have requires
_grad=True. When you call
loss.backward(), it automatically computes the gradient of the loss with respect to every
parameter in the network. You do not write the gradient equations by hand.
PyTorch concept What it does
nn.Module Base class for all neural network layers and models
nn.Linear(in, out) Fully connected layer: output = input @ W + b
F.relu(x) Rectified Linear Unit activation: max(0, x)
F.softmax(x, dim=-1) Converts logits to probabilities that sum to 1
torch.log(prob) Natural logarithm — used for log pi(a|s)
dist.Categorical(probs) Discrete probability distribution for sampling
dist.log_prob(action) log pi(a|s) for the sampled action
optimizer.zero_grad() Clear gradients from the previous update
loss.backward() Compute gradients via backpropagation
optimizer.step() Update network weights using the computed gradients

6. Understanding the Gradient Update (Conceptual
Walkthrough)
This section walks through what happens numerically in one policy update so you
understand what PyTorch is computing behind the scenes.
Suppose after episode 1 the agent took 3 steps before the pole fell:
Step 0: state=[0.01, 0.02, -0.01, 0.03], action=Right, reward=+1
Step 1: state=[0.02, 0.14, -0.02, -0.10], action=Left, reward=+1
Step 2: state=[0.05, -0.05, -0.04, 0.15], action=Right, reward=+1
(pole fell — episode over)
Step A — Compute returns (gamma=0.99)
G_2 = 1.0
G_1 = 1 + 0.99 * 1.0 = 1.99
G_0 = 1 + 0.99 * 1.99 = 2.9701
Step B — Normalise returns
mean(G) = (2.9701 + 1.99 + 1.0) / 3 = 1.987
std(G) = 0.807 (approx)
G_0_norm = (2.9701 - 1.987) / 0.807 = +1.218
G_1_norm = (1.990 - 1.987) / 0.807 = +0.004
G_2_norm = (1.000 - 1.987) / 0.807 = -1.223

Step C — Compute log probabilities
Suppose the policy output these probabilities at each step:
Step 0: pi(Left)=0.45, pi(Right)=0.55 => action=Right => log(0.55) =
-0.598
Step 1: pi(Left)=0.52, pi(Right)=0.48 => action=Left => log(0.52) =
-0.654
Step 2: pi(Left)=0.49, pi(Right)=0.51 => action=Right => log(0.51) =
-0.673
Step D — Compute the REINFORCE loss
loss = -[ G_0_norm * log_pi_0 + G_1_norm * log_pi_1 + G_2_norm *
log_pi_2 ]
loss = -[ 1.218*(-0.598) + 0.004*(-0.654) + (-1.223)*(-0.673) ]
loss = -[ -0.728 + -0.003 + 0.823 ]
loss = -[ 0.092 ]
loss = -0.092
PyTorch calls loss.backward() and computes how each weight in the network contributed to
this loss. The Adam optimizer then nudges the weights to make the loss more negative
(which means making it more positive before the negation — i.e. increasing the probability of
good actions).
The first episode is tiny (3 steps). As the policy improves and episodes get longer, the
magnitude of the returns and the loss both increase — which is why loss values grow with
episode length.
